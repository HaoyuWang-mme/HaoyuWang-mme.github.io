## 开始写代码：你的第一个Llama调用

现在我们来写一段Java代码，用千聚API调用Llama 3模型。这段代码会完成一个最简单的“对话”任务——问Llama：“你好，你是谁？”

我们先创建一个普通的Maven Java项目（IDE不限，IDEA、Eclipse、Visual Studio Code都行）。

首先，在 `pom.xml` 里引入一个核心依赖：**OkHttp**。它比Java原生的HttpClient好写很多，而且性能更好、更稳定。

xml
<dependency>
    <groupId>com.squareup.okhttp3</groupId>
    <artifactId>okhttp</artifactId>
    <version>4.12.0</version>
</dependency>


然后，在 `src/main/java` 下创建一个类，比如就叫 `LlamaDemo.java`。

java
import okhttp3.*;
import com.google.gson.*;
import java.io.IOException;

public class LlamaDemo {
    public static void main(String[] args) throws IOException {
        // 1. 替换成你自己的API Key
        String apiKey = "sk-你的APIKey"; 
        // 2. 千聚API的统一接口地址
        String url = "https://www.qianjuai.com/v1/chat/completions";

        // 3. 构建请求体（OpenAI兼容格式，Llama也支持）
        JsonObject message = new JsonObject();
        message.addProperty("role", "user");
        message.addProperty("content", "你好，你是谁？");

        JsonArray messages = new JsonArray();
        messages.add(message);

        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", "llama3-8b-8192"); // 这里用Llama 3 8B模型
        requestBody.add("messages", messages);
        requestBody.addProperty("temperature", 0.7);
        requestBody.addProperty("max_tokens", 200);

        // 4. 使用OkHttp发送请求
        OkHttpClient client = new OkHttpClient();
        MediaType JSON = MediaType.get("application/json; charset=utf-8");
        Request request = new Request.Builder()
                .url(url)
                .header("Authorization", "Bearer " + apiKey)
                .post(RequestBody.create(requestBody.toString(), JSON))
                .build();

        // 5. 接收响应并输出
        try (Response response = client.newCall(request).execute()) {
            String body = response.body().string();
            // 解析JSON，提取回复内容
            JsonObject jsonResponse = JsonParser.parseString(body).getAsJsonObject();
            String reply = jsonResponse.getAsJsonArray("choices")
                              .get(0).getAsJsonObject()
                              .getAsJsonObject("message")
                              .get("content").getAsString();
            System.out.println("Llama回复: " + reply);
        }
    }
}


你只需要把 `apiKey` 变量替换成你从千聚后台复制的Key，然后直接运行 `main` 方法就行了。

## 代码解读：每一步都在做什么？

这段代码看起来有点长，其实只做五件事：

1. **设置接口地址和授权** —— 使用千聚的统一域名 `www.qianjuai.com/v1`，然后加上 `Authorization` 头。
2. **构建请求体** —— 符合OpenAI的 `/chat/completions` 格式。这里指定模型为 `llama3-8b-8192`，这是千聚支持的Llama 3模型ID之一。你也可以换成 `llama3-70b-8192` 或者 `llama3.1-8b`，前提是检查模型列表确认被支持。
3. **发送请求** —— 用OkHttp的post方法，指定JSON类型。
4. **接收响应** —— 同样接收一个JSON。
5. **提取内容** —— 从 `choices[0].message.content` 字段里拿到Llama的回复文本。

就这么简单。运行成功后，你会在控制台看到Llama回复的一小段自我介绍。大功告成，第一个Java调用Llama的示例就跑通了。

## 常见避坑全图（小白必看）

这10分钟里，大部分人遇到的问题是下面这几种。我一次性列清楚，你遇到了直接按着解决。

**1. “Exception: 403 Forbidden” 或 “401 Unauthorized”**

这是最常见的。原因只有一个：API Key写错了，或者Key过期了。解决办法：重新去千聚后台复制一次Key，粘贴进代码时确认没有多余的空格或换行。注意是`Bearer sk-xxx`，不是别的。

**2. “No model found with name llama3-8b-8192”**

你用的模型ID千聚不支持。去官网查一下最新的模型列表。Llama 3 常见的支持ID有：`llama3-8b-8192`、`llama3-70b-8192`、`llama3.1-8b`。如果都不行，可以改成 `gpt-3.5-turbo` 先测试接口通不通，确认没问题再换回Llama的ID。

**3. 使用Java原生的HttpURLConnection报各种奇怪的SSL错误**

如果你的JDK版本较低（比如JDK 8），并且用原生API，可能会遇到SSL验证失败。最好的办法就是换成OkHttp，或者直接升级到JDK 17。OkHttp帮你处理了绝大多数底层问题。

**4. 代码正常返回，但内容是空的**

检查一下 `max_tokens` 是不是设得太低（比如小于10）。建议设成200以上，Llama生成回复需要一些空间。

**5. “Connect timed out” 或 “No route to host”**

国内网络环境问题，少数情况可能发生。但千聚是国内直连，理论上不会。如果出现，去千聚官网确认节点状态，或者重启一下本地网络环境。

**6. 不知道怎么把回复解析成中文不乱码**

确保你的代码文件编码是UTF-8。IDEA里默认是UTF-8，但Eclipse或者文本编辑器可能需要手动设置。在 `pom.xml` 里加一句：

xml
<properties>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
</properties>
