### 2. 文本对话示例（非视觉类）

如果你只是需要Qwen的文本理解能力，调用方式跟GPT-3.5完全一样：

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.util.ArrayList;
import java.util.List;

public class QwenTextDemo {
    public static void main(String[] args) {
        // 核心：将OPENAI的baseUrl替换为千聚的地址
        OpenAiService service = new OpenAiService(
                "sk-你的千聚API密钥", // 从千聚获取的API Key
                60 // 超时时间（秒）
        );

        // 设置baseUrl
        service.setBaseUrl("https://www.qianjuai.com/v1");

        // 构建对话请求
        List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage("user", "请用Java写一个简单的工厂模式代码示例"));

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("qwen-vl-max") // 指定千问视觉模型
                .messages(messages)
                .maxTokens(1024)
                .temperature(0.7)
                .build();

        // 发送请求并打印回复
        service.createChatCompletion(request).getChoices().forEach(
                choice -> System.out.println(choice.getMessage().getContent())
        );
    }
}


**关键点解析**：
- `setBaseUrl("https://www.qianjuai.com/v1")`：这就是千聚让你“省事”的地方——一行代码，把原来指向OpenAI的地址改成了千聚的地址。
- `model("qwen-vl-max")`：直接写千问的模型名，千聚会自动帮你路由到阿里官方模型。

### 3. 图像理解示例（视觉类）

Qwen-VL的视觉能力非常强大，下面是一段“看图说话”的Java代码：

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageContent;
import com.theokanning.openai.completion.chat.ChatMessageContentPart;
import java.util.ArrayList;
import java.util.Base64;
import java.util.List;

public class QwenVisionDemo {
    public static void main(String[] args) {
        OpenAiService service = new OpenAiService(
                "sk-你的千聚API密钥",
                120 // 视觉任务可能需要更长超时
        );
        service.setBaseUrl("https://www.qianjuai.com/v1");

        // 准备图片数据（URL或Base64）
        // 方式A：使用图片URL
        String imageUrl = "https://www.qianjuai.com/register";
        
        // 方式B：使用Base64编码（从本地文件读取并编码）
        // String base64Image = encodeImageToBase64("/path/to/local/image.png");
        // String imageUrl = "data:image/png;base64," + base64Image;

        // 构建支持多模态的消息
        List<ChatMessageContentPart> parts = new ArrayList<>();
        parts.add(new ChatMessageContentPart("text", "请描述这张图片的内容。"));
        parts.add(new ChatMessageContentPart("image_url", imageUrl));

        ChatMessageContent content = new ChatMessageContent();
        content.setParts(parts);

        List<ChatMessageContent> contents = new ArrayList<>();
        contents.add(content);

        ChatMessage userMessage = new ChatMessage();
        userMessage.setRole("user");
        userMessage.setContent(contents);

        List<ChatMessage> messages = new ArrayList<>();
        messages.add(userMessage);

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("qwen-vl-max")
                .messages(messages)
                .maxTokens(2048)
                .temperature(0.8)
                .build();

        // 发送请求并打印回复
        service.createChatCompletion(request).getChoices().forEach(
                choice -> System.out.println(choice.getMessage().getContent())
        );
    }
    
    // 辅助方法：将图片文件编码为Base64
    private static String encodeImageToBase64(String imagePath) {
        try {
            byte[] imageBytes = java.nio.file.Files.readAllBytes(java.nio.file.Paths.get(imagePath));
            return Base64.getEncoder().encodeToString(imageBytes);
        } catch (Exception e) {
            e.printStackTrace();
            return "";
        }
    }
}


**重要提示**：如果你的图片很大（比如超过10MB），建议先用工具压缩，或者使用图片URL方式提交，避免请求超时。

## 常见报错与解决指南

在实际运行过程中，你可能会遇到以下几种情况，这里直接给出解决方案：

### 错误一：`401 Unauthorized` 或 `Authentication Fails`

**原因**：API Key不对或未正确设置。

**解决方法**：
- 去千聚控制台重新复制API Key，确保没有多余空格。
- 检查你Java代码里`setApiKey()`或构造函数`new OpenAiService("sk-...")`里的值是否正确。

### 错误二：`404 Not Found` 或 `Model not found`

**原因**：你指定的模型名称在千聚上不存在或未激活。

**解决方法**：
- 确保你用的是`qwen-vl-max`、`qwen-vl-plus`或`qwen-vl-lite`。如果是最新的Qwen2.5-VL系列，请确认千聚上显示的准确名称。
- 登录控制台，在“模型列表”中搜索“qwen”，确认你需要的模型支持状态。

### 错误三：超时（`TimeoutException` 或 `connect timed out`）

**原因**：网络连接问题，或图片文件过大导致处理时间过长。

**解决方法**：
- 检查你能否正常访问`https://www.qianjuai.com/v1`。如果不能，先检查代理设置（关闭代理后重试）。
- 如果是视觉任务，增加超时时间：将`new OpenAiService(apiKey, **120**)`。
- 使用图片URL代替Base64编码，因为Base64编码后的字符串可能非常大，导致传输耗时。

### 错误四：`400 Bad Request` - `Invalid `messages`

**原因**：消息格式不对，视觉消息的构造不符合OpenAI规范。

**解决方法**：
- 务必使用带有`parts`列表的`ChatMessageContent`类型来构建多模态消息，而不是简单的字符串。
- 参考上面“图像理解示例”中的结构，确保参数正确。
