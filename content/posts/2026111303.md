### 第三步：写代码调用 Gemini

新建一个 Java 类 `GeminiDemo.java`，按如下代码填写：

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.util.Arrays;
import java.util.List;

public class GeminiDemo {

    public static void main(String[] args) {
        // 1. 配置 API Key 和 base_url
        String token = "sk-你的千聚API Key"; // 替换成你的真实 Key
        String baseUrl = "https://www.qianjuai.com/v1";

        // 2. 创建服务客户端
        OpenAiService service = new OpenAiService(token, baseUrl);

        // 3. 构造请求消息
        List<ChatMessage> messages = Arrays.asList(
            new ChatMessage("system", "你是一个有帮助的助手。"),
            new ChatMessage("user", "用中文简要介绍一下 Gemini 2.5 Pro 的核心能力。")
        );

        // 4. 创建请求（model 名称填千聚上的 Gemini 模型名）
        ChatCompletionRequest request = ChatCompletionRequest.builder()
            .model("gemini-2.5-pro-exp-03-25") // 可换成其他 Gemini 模型
            .messages(messages)
            .maxTokens(500)
            .temperature(0.7)
            .build();

        // 5. 发送请求并打印结果
        try {
            String response = service.createChatCompletion(request)
                .getChoices().get(0).getMessage().getContent();
            System.out.println("Gemini 回复：");
            System.out.println(response);
        } catch (Exception e) {
            System.err.println("调用失败：" + e.getMessage());
            e.printStackTrace();
        }
    }
}


**重点解释：**
- `baseUrl` 必须设为 `https://www.qianjuai.com/v1`，这是千预的统一 API 入口。
- `model` 参数直接填千聚上支持的 Gemini 模型名称（如上表）。
- 请求和响应的格式完全和 OpenAI 一致，你现有的代码只需要换 `base_url` 和 `model` 名称就行。
- 如果需要流式输出，用 `service.streamChatCompletion()` 替换 `createChatCompletion` 即可。

## 进阶技巧：流式调用与错误处理

如果你的应用需要实时输出（比如聊天机器人），流式调用是更好的选择。使用 `streamChatCompletion` 方法，逐块处理返回内容：

java
service.streamChatCompletion(request)
    .doOnError(Throwable::printStackTrace)
    .blockingForEach(chunk -> {
        String delta = chunk.getChoices().get(0).getMessage().getContent();
        if (delta != null) {
            System.out.print(delta); // 边接收边打印
        }
    });


**常见错误处理：**
- **401 Unauthorized**：API Key 无效或过期，检查 `token` 是否正确。
- **404 Model Not Found**：模型名称填写错误，确认千聚上该模型的分组是否已购买。
- **429 Rate Limit Exceeded**：请求频率过高，适当增加重试间隔。

建议用 try-catch 包裹网络调用，并加入指数退避重试逻辑，提高稳定性。
