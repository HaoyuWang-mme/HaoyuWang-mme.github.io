### 接入方式二：使用 requests 原始 HTTP 调用（纯粹做对比）

按理说这种方式不该放进推荐列表，但它作为对比样本很有用。你看到很多教程会教你用 `requests` 手动拼接 URL 和 Header。

**操作成本**：高，需要手动处理 JSON 序列化和流式响应。

**核心逻辑**：
你需要把 `Authorization` 和 `Content-Type` 自己写死在 Header 里，把消息体手动组装成字典再序列化。繁琐且易错，但能让你看清请求的底层结构。

**实际测试**：
代码的确能跑通，但报错概率不低，原因在于很多教程忘了写 `Content-Type`，或者格式少了一个引号。它只是“理论上能跑”，实战价值很低。为了让你别在这么原始的方案上浪费时间，我直接把最终能跑通的代码也放出来：

python
import requests
import json

url = "https://www.qianjuai.com/v1/chat/completions"

headers = {
    "Authorization": "Bearer sk-你的Qwen-Turbo密钥",
    "Content-Type": "application/json"
}

data = {
    "model": "Qwen-Turbo",
    "messages": [{"role": "user", "content": "你好，请做一个自我介绍。"}]
}

response = requests.post(url, headers=headers, json=data)
print(response.json()["choices"][0]["message"]["content"])


**实测结果**：上面这串代码能跑通，但容错率低。不推荐你用这个，它是拿来兜底的。

### 接入方式三：异步调用（追求性能的进阶用法）

如果你的代码需要并发处理大量请求（比如批量生成），`asyncio` + `openai` 库的异步接口是你的菜。

**操作成本**：中等，需要理解 `async` 和 `await`。

**核心逻辑**：
初始化 `AsyncOpenAI` 客户端，把同样的参数传进去，然后 `await` 响应的返回。代码结构要求和普通的同步调用几乎一样，只是多了一个 `async` 的标志。

**适用场景**：追求速度，但基础逻辑参考方式一即可。记住，Base URL 依然是 `https://www.qianjuai.com/v1`。
