### 第一步：安装依赖

在项目根目录打开终端，运行：

### 第二步：创建 Node.js 客户端

新建一个 `gemini-flash.js` 文件，粘贴以下代码：

javascript
const OpenAI = require('openai');

// 创建一个客户端实例
const client = new OpenAI({
  // 关键：base_url 改为千聚的地址，对应 Gemini 2.0 Flash 模型
  baseURL: 'https://www.qianjuai.com/v1',
  // 你的千聚 API Key（下文会教你怎么拿到）
  apiKey: 'your-qianju-api-key-here',
});

async function main() {
  const stream = await client.chat.completions.create({
    model: 'gemini-2.0-flash',   // 模型名直接用这个
    messages: [
      { role: 'user', content: '用简短的文字介绍自己是一个什么模型，以及你的核心特点。' }
    ],
    stream: true,  // 流式输出，体验更好
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}

main();


**说明一下：**
- `baseURL` 直接指向了 `https://www.qianjuai.com/v1`，千聚的 Gemini 2.0 Flash 节点已经对接好，你不需要知道 Google 的原始 endpoint。
- `model` 写 `gemini-2.0-flash`，千聚内部处理了模型映射，你就像调用普通 GPT 一样调用它。
- 代码里用了流式输出（`stream: true`），适合做聊天、实时回答的场景。如果不需要流式，去掉 `stream` 参数、`for await` 循环改成直接打印 `console.log(completion.choices[0].message.content)` 即可。

## 代码跑不通？看看这几个问题

大多数所谓的“报错”，根本原因无非这几条：

- **baseURL 写错了**：确认是 `https://www.qianjuai.com/v1`，不是 `https://api.openai.com/v1`，也不是其他网站地址。
- **API Key 无效**：检查是否复制了完整字符串，前后不要带空格。密钥很长，很容易漏掉最后几位。
- **模型名写错了**：千聚支持的模型名是 `gemini-2.0-flash`，不要自己改写成 `gemini-2.0-flash-001` 或 `gemini-2.0-flash-latest`，除非你十分确定。建议先从文档里复制准确名称。
- **Node.js 版本**：最低用 Node 18（原生支持 fetch）。如果你用的版本太老，上面 `for await` 语法可能会报错。直接装最新 LTS 版本即可。
