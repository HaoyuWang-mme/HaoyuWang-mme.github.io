## 第四步：玩点高级的——传入图片（多模态）

Gemini 的多模态能力是一大亮点，能直接识图、分析图片内容。[千聚api中转站](https://www.qianjuai.com/)也完全支持这个功能。

下面的代码演示了如何传入一张本地图片，让 Gemini 描述图片内容：

python
import base64
from openai import OpenAI

client = OpenAI(
    base_url="https://www.qianjuai.com/v1",
    api_key="sk-你的千聚APIKey"
)

# 读取图片并编码为base64
def encode_image(image_path):
    with open(image_path, "rb") as image_file:
        return base64.b64encode(image_file.read()).decode('utf-8')

image_path = "你的图片路径.jpg"
base64_image = encode_image(image_path)

response = client.chat.completions.create(
    model="gemini-2.0-flash",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "请详细描述这张图片里有什么？"},
                {
                    "type": "image_url",
                    "image_url": {
                        "url": f"data:image/jpeg;base64,{base64_image}"
                    }
                }
            ]
        }
    ]
)

print(response.choices[0].message.content)


这种方式非常适合做商品图分析、图像审核、截图问答等场景。代码结构和OpenAI的视觉API完全一致，熟悉OpenAI的话零学习成本。

## 第五步：调整参数，让模型听话

你可以通过调整参数来控制Gemini的回复风格。常用的参数是 `temperature`（温度）和 `max_tokens`（最大输出长度）。

python
response = client.chat.completions.create(
    model="gemini-2.0-flash",
    messages=[
        {"role": "user", "content": "写一首关于秋天的短诗"}
    ],
    temperature=0.7,  # 值越高，回复越有创意，0.0-2.0之间
    max_tokens=200   # 限制回复的字数
)


- `temperature`: 控制结果的随机性。值越低（如0.2），模型越严谨、遵循事实；值越高（如1.0），模型越有创造力和想象力。
- `max_tokens`: 限制模型生成的最大token数，可以防止回复过长，控制成本。
