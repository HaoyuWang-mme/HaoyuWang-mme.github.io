#### 第四部分：踩坑记录 —— 你的“一场大病”是他的“预防针”

这部分是**最值钱**的部分。一定要真实地记录你遇到的问题，以及如何解决的。

**具体踩坑范例**：

-   **坑1：HTTP 502 错误。**
    -   **症状**：运行代码时，返回 `Error: 502 Bad Gateway`。
    -   **原因分析**：我一开始没注意分组。我在代码里指定了 `model=“claude-3-sonnet”`，但我用的是“默认（混合）”分组。千聚API 的分组里，“默认”组针对 Claude 有单独的收费，但请求量突增时，可能会出现这个短期错误。
    -   **解决方案**：要么换模型，比如换成 `gpt-4o`，因为“默认”分组对 OpenAI 的 Azure 渠道很稳定（1:1费率）；要么去后台切换到“官转克劳德 2”分组（3x费率），专门用于稳定调用 Claude，但贵几倍。**结论**：你先用 `gpt-4o` 试水，不要一上来就挑战最贵最复杂的模型。
-   **坑2：API Key 格式报错。**
    -   **症状**：提示 `AuthenticationError`。
    -   **原因分析**：我把直接在千聚后台生成的 API Key 复制错了，多复制了一个空格。或者是，我直接复制了OpenAI的旧Key过来，格式不匹配。
    -   **解决方案**：解码方案很简单。**重新生成一个干净的 Key**，用 `Ctrl+V` 粘贴到代码里，粘贴后**双击一下，确保没有两端空白字符**。千聚API的Key都是固定格式，检查你粘贴进去的是不是 `sk-xxx` 开头。
