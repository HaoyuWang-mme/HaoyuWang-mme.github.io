### 第三步：Java客户端配置类

java
@Configuration
public class MistralClientConfig {

    @Value("${mistral.api.base-url}")
    private String baseUrl;

    @Value("${mistral.api.api-key}")
    private String apiKey;

    @Bean
    public OpenAiService openAiService() {
        return new OpenAiService(baseUrl, apiKey, Duration.ofSeconds(30));
    }
}


### 第四步：实现业务接口（Controller + Service）

java
@RestController
@RequestMapping("/api/chat")
public class MistralChatController {

    @Autowired
    private MistralApiService mistralApiService;

    @PostMapping("/completion")
    public ChatResponse chat(@RequestBody ChatRequest request) {
        return mistralApiService.callMistral(request);
    }
}


Service层是真干活的地方：

java
@Service
public class MistralApiService {

    @Autowired
    private OpenAiService openAiService;

    @Value("${mistral.api.model}")
    private String model;

    public ChatResponse callMistral(ChatRequest request) {
        CompletionRequest completionRequest = CompletionRequest.builder()
                .model(model)
                .messages(request.getMessages())
                .maxTokens(1024)
                .temperature(0.7)
                .build();

        ChatCompletionResult result = openAiService.createChatCompletion(completionRequest);

        ChatResponse response = new ChatResponse();
        response.setReply(result.getChoices().get(0).getMessage().getContent());
        response.setTokenUsage(result.getUsage().getTotalTokens());
        return response;
    }
}


这里调用的就是Mistral的原生能力。但你看看代码里的 `model` 字段——如果明天你想换Claude或GPT，配置文件里改成 `claude-sonnet-4-20250514` 或 `gpt-4o`，一行不动其他代码就能切换。千聚支持500+模型，全都在同一个API路径下。
