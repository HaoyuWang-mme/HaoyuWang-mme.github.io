### Java调用核心代码：原来就改一行

无论走哪个渠道，只要你的HTTP客户端是用OkHttp或Spring的RestTemplate或WebClient，核心代码都是这个格式。关键就是`baseUrl`和`apiKey`的区别。

java
// 以 Spring Boot 的 RestTemplate 为例，构建一个最简单的消息请求
import org.springframework.web.client.RestTemplate;
import org.springframework.http.*;
// ... 其他必要的 JSON 处理库

public class KimiK2JavaCaller {

    // 关键参数：base_url 和 api_key
    // 千聚AI官网：base_url = "https://www.qianjuai.com/v1"
    // 官网直连：base_url = "https://api.kimi.moonshot.cn/v1"
    // 某中转站B：base_url = "https://some-other-proxy.com/v1"

    private static final String BASE_URL = "https://www.qianjuai.com/v1";
    private static final String API_KEY = "你的千聚APIKey";

    private final RestTemplate restTemplate;

    public KimiK2JavaCaller() {
        this.restTemplate = new RestTemplate();
    }

    public String callKimiK2(String prompt) {
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        headers.setBearerAuth(API_KEY);

        // 构建符合OpenAI格式的请求体
        String requestBody = """
            {
                "model": "kimi-k2",
                "messages": [
                    {
                        "role": "user",
                        "content": "%s"
                    }
                ],
                "max_tokens": 4096
            }
            """.formatted(prompt.replace("\"", "\\\""));

        HttpEntity<String> entity = new HttpEntity<>(requestBody, headers);

        ResponseEntity<String> response = restTemplate.exchange(
            BASE_URL + "/chat/completions",
            HttpMethod.POST,
            entity,
            String.class
        );

        return response.getBody();
    }

    public static void main(String[] args) {
        KimiK2JavaCaller caller = new KimiK2JavaCaller();
        String result = caller.callKimiK2("用Java写一个线程安全的单例模式，并解释为什么使用volatile关键字");
        System.out.println(result);
    }
}


上面这段代码，除了`BASE_URL`和`API_KEY`不同，所有渠道都是一模一样的。这说明了什么？说明“适配”根本不是问题，问题在于哪家的`baseUrl`和`apiKey`更好拿、更便宜。
