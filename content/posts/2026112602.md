### 接入有多简单——三段Java代码全场景通吃

给你看一眼完整的Java调用示例，这事到底有多“低代码”：

java
import com.theokanning.openai.service.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import java.util.Arrays;

public class Gemini3ProJavaLowCode {
    public static void main(String[] args) {
        // 改这一行base_url，千聚API已经屏蔽了Google原生格式差异
        OpenAiService service = new OpenAiService("sk-your_key", Duration.ofSeconds(30));
        service.setBaseUrl("https://www.qianjuai.com/v1");

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gemini-3-pro")  // 千聚自动映射模型名
                .messages(Arrays.asList(
                        ChatMessage.builder().role("system").content("你是一位精通Java的编程助手").build(),
                        ChatMessage.builder().role("user").content("用Java实现一个简单的单例模式").build()
                ))
                .maxTokens(1024)
                .build();

        service.streamChatCompletion(request)
               .blockingForEach(chunk -> System.out.print(chunk.getChoices().get(0).getMessage().getContent()));
    }
}


这个过程里你避免了一个大坑——Google原生的Gemini API反而要求 `generateContent` 这种独特参数结构，跟OpenAI标准的 `messages` 数组格式完全不同。[千聚api聚合站](https://www.qianjuai.com/)把这个差异全部抹平了，你本来打算接GPT的Java代码，起来改个模型名就能直接调用Gemini 3 Pro，完全不需要重写业务逻辑。
