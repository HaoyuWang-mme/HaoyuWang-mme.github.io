### “网络错误”到底怎么一回事？先搞明白再动手

用 Node.js 调用 GPT-5pro，本质上就是让你的代码去“敲门”——敲 OpenAI API 服务器的门。如果你的代码在终端里报了 `fetch failed`，说明你的“敲门”请求在半路上被人拦住了，或者根本没敲门就超时了。

那为什么会出现这个问题？两个主要原因：

1. **直接访问海外地址受阻**：Node.js 默认请求 `https://api.openai.com`，这个地址在国内没有正常直连通道。
2. **代理配置混乱或缺失**：很多开发者知道要设置代理，但不知道具体填哪个地址、填到哪里。特别是用 `openai` Node SDK 时，代理设置稍微写错一个字符，可能就会导致握手失败。

**本质上，你只需要把“敲门的对象”从海外换成一个国内能访问的中转站**。不要自己去折腾各种翻墙方案，那是一条弯路。

### 真有一行能搞定？代码对比来了

因为你只改了“敲门地址”，没改“敲门方式”。**node-openai SDK 默认会以特定方式拼接路由，如果你只替换基础地址，不做接口完全匹配，断然不行。**

真正的一行“救急”代码不是简单的 URL 替换，而是**同时配置基础路径和关闭不必要的默认验证选项**。
