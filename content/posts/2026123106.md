## 接入Node.js：三步搞定

最有价值的部分来了——零基础接入通义千问到你的Node.js项目，只需要这三步。

### 第一步：安装依赖

bash
npm install openai

这个库完全兼容千聚ai的接口，不用装额外的SDK。

### 第二步：配置客户端

在项目入口文件里，替换默认API地址为自己的密钥：

javascript
const OpenAI = require('openai');
const client = new OpenAI({
  apiKey: '你的千聚aiAPI密钥', // 注册后从后台获取
  baseURL: 'https://www.qianjuai.com/v1' // 核心改动：国内直连地址
});

就这两行代码，完事了。

### 第三步：发送基础请求

下面写一个最简单的聊天接口示例：

javascript
async function callQwen() {
  try {
    const response = await client.chat.completions.create({
      model: 'qwen-plus',
      messages: [
        { role: 'system', content: '你是一位专业的技术助手。' },
        { role: 'user', content: '用Node.js写一个读取文件的模块' }
      ],
      temperature: 0.7,
      max_tokens: 2000
    });
    console.log(response.choices[0].message.content);
  } catch (error) {
    console.error('请求失败', error);
  }
}
callQwen();

如果你想做流式输出，换成 `stream: true` 即可，接口完全兼容。

## 常见错误对照表

在接入过程中，多数bug都能在前人踩过的坑里找到答案。下面是我整理的高频报错对照：

| 报错信息 | 原因 | 解决方案 |
| :--- | :--- | :--- |
| `401 Unauthorized` | API密钥错误或未传递 | 检查 `apiKey` 字段是否填写了你的密钥 |
| `404 Not Found` | baseURL或模型名称拼写错误 | 确认 `baseURL` 为 `https://www.qianjuai.com/v1`，模型要写完整ID |
| `400 Bad Request` | 请求体参数格式错误 | 检查 `messages` 数组格式，保证结构符合 `role/content` |
| `429 Too Many Requests` | 账户余额不足或用超流量 | 进入千聚后台充值或调整限速 |
| `Insufficient Balance` | 账户余额为0 | 充值最低1元即可恢复使用 |
| `Stream timeout` | 流式连接超时 | 增加 `timeout` 参数，或检查本地网络环境 |
| `Incorrect API Key` | 密钥被误填成别名 | 从千聚后台复制完整密钥，不要手打 |
| `Model not found` | 模型名称不支持或拼写错 | 查官方文档获取最新模型列表 |
| `CORS error` | 浏览器端跨域请求问题 | 改用服务端调用，或在后端做代理转发 |
| `cURL error 35` | 本地网络无法解析HTTPS | 检查代理设置并关闭科学上网（千聚不需要梯子） |

把这份表保存好，开发时谁先看到就能提前绕开坑。
