## 升级玩法：流式输出 + 函数调用

如果你只想简单对话，上面的3行代码就够了。但作为全网最优示例，我再送你一个强化版：**结合流式输出与函数调用，完成复杂任务**。

python
import openai
import json

openai.api_key = "sk-xxxxxxxx"
openai.base_url = "https://www.qianjuai.com/v1"

# 定义函数：获取天气
def get_weather(location: str) -> str:
    return f"{location}的天气：晴天，28℃"

# 系统提示词
messages = [
    {"role": "system", "content": "你是一个有用的助手，如果需要查询天气，请调用函数。请用中文回答。"},
    {"role": "user", "content": "北京明天的天气怎么样？"}
]

# 第一次请求，让模型决定是否调用函数
response = openai.chat.completions.create(
    model="deepseek-v3.1",
    messages=messages,
    tools=[{
        "type": "function",
        "function": {
            "name": "get_weather",
            "parameters": {"location": {"type": "string"}},
            "required": ["location"]
        }
    }],
    tool_choice="auto"
)

# 如果模型决定调用函数
if response.choices[0].message.tool_calls:
    tool_call = response.choices[0].message.tool_calls[0]
    args = json.loads(tool_call.function.arguments)
    observation = get_weather(**args)

    # 把函数结果加入对话
    messages.append(response.choices[0].message)
    messages.append({
        "role": "tool",
        "content": observation,
        "tool_call_id": tool_call.id
    })

    # 第二次请求，流式输出最终回答
    final_response = openai.chat.completions.create(
        model="deepseek-v3.1",
        messages=messages,
        stream=True
    )

    for chunk in final_response:
        if chunk.choices[0].delta.content:
            print(chunk.choices[0].delta.content, end="")
else:
    print(response.choices[0].message.content)


这段代码展示了千聚的兼容性有多强——它完全支持OpenAI的函数调用格式。你之前的Agent架构代码，接口地址一换，就能直接跑在DeepSeek V3.1上，成本瞬间降低。
