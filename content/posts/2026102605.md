## 第三步：数据解析——分场景处理

### 非流式（上面就是）

返回的是一个标准的 OpenAI completion 对象：`completion.choices[0].message.content` 拿到文本。`completion.usage` 包含tokens消耗。

### 流式（Stream）

如果要做实时输出，加上 `stream: true`：

javascript
const stream = await client.chat.completions.create({
  model: 'ernie-bot',
  messages: [{ role: 'user', content: '讲个笑话' }],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '');
}


千聚中转站完全支持SSE流式，不用改任何逻辑。需要注意的是，百度的原生流式返回格式不一样，但经过千聚转换后，你收到的数据跟OpenAI API一模一样，所以用现成的库解析没问题。
