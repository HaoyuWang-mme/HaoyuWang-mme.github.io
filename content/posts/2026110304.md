## 第三步：调用报错全排雷——你可能会遇到的5个坑

这是全文最值钱的部分。我把新手最常遇到的几个报错和原因，一次性给你讲明白。

### 坑1: 状态码 401 - Unauthorized（未授权）

**现象：** 代码运行时，返回 `401 Unauthorized` 错误。
**原因：** 90%的情况是你的 `api_key` 写错了，或忘了替换代码中的占位符。
**解决方案：**
1.  检查你的Key是否是从[千聚api聚合站](https://www.qianjuai.com/)复制的最新Key。
2.  检查代码中的 Key 是否被双引号包裹正确。
3.  如果你的Key有空格或换行符，请彻底清除。

### 坑2: 状态码 404 - Not Found（模型不存在）

**现象：** 返回 `404 Not Found`。
**原因：** 你代码里写的模型名（`model` 参数）不存在或拼写错误。比如，你把 `qwen-plus` 写成了 `qwen-plus-xxx` 或者 `qwen-PLUS`（注意大小写）。
**解决方案：**
1.  确保模型名称严格按[千聚api聚合站](https://www.qianjuai.com/)提供的列表里的名称填写，去官网查看最新的模型列表。

### 坑3: 状态码 429 - Rate Limit Exceeded（请求频率过高被限速）

**现象：** 代码跑得过快，连续调用几十次后突然报这个错。
**原因：** 触发了平台的速率限制（Rate Limit），防止滥用。这通常是正常的保护机制。
**解决方案：**
1.  在你的代码里，在每次请求之间加入一个短暂的睡眠时间，比如 `time.sleep(0.5)`。
2.  如果是生产环境高并发，请联系客服申请提高速率限制。

### 坑4: 状态码 503 - Service Unavailable（暂时无响应）

**现象：** 调用时偶尔报 `503`。
**原因：** 模型源站（比如阿里云）或中转节点偶发性不可用。
**解决方案：**
1.  这是正常现象，几乎任何API平台都会出现，属于偶发错误。
2.  **最佳实践：** 在你的代码中加入重试机制（比如 `tenacity` 库），失败后等待几秒自动重试1-2次。通常一次重试就能成功。

### 坑5: 流式响应（Streaming）报错

**现象：** 使用 `stream=True` 参数时，返回的流被打断或报错。
**原因：** 通常是你处理流式响应的方法不对，或者在流式传输过程中网络抖动。
**解决方案：**
1.  确保你的代码正确迭代流式对象。
2.  确保你在代码的 `stop` 参数或 `finish_reason` 为 `stop` 时正确处理结束。很多人在打印或解析流式数据时漏掉了字段。

**核心心法：** 当你遇到任何报错，第一反应不是慌，而是打开浏览器，刷新[千聚api聚合站](https://www.qianjuai.com/)的状态页面，确认是不是平台整体故障。如果平台稳妥，就回头检查你的代码。80%的bug都是Key写错或模型名写错。

## 实战案例：10分钟写一个你自己的AI助手

学了这么多，我们来做一个最小的、能用的东西。

假设你想写一个命令行智能助手。

python
import openai
import sys

client = openai.OpenAI(
    base_url="https://www.qianjuai.com/v1",
    api_key="sk-你的千聚API Key"
)

def chat_with_ai():
    print("你好，我是你的AI助手（输入 'exit' 退出）")
    
    while True:
        user_input = input("\n你: ")
        
        if user_input.lower() == 'exit':
            break
        
        try:
            response = client.chat.completions.create(
                model="qwen-turbo",
                messages=[
                    {"role": "server", "content": "你是用户最贴心的助手，回答问题简洁、准确。"},
                    {"role": "user", "content": user_input}
                ],
                # 安全地处理流式输出
                stream=True
            )
            
            print("AI: ", end="")
            for chunk in response:
                if chunk.choices[0].delta.content is not None:
                    print(chunk.choices[0].delta.content, end="")
            print("")  # 换行

        except openai.APIConnectionError as e:
            print(f"\n【错误提示】连接失败：{e}")
            print("请检查你的网络或API地址是否正确。")
        except openai.RateLimitError:
            print("\n【错误提示】请求过于频繁，请稍后再试。")

if __name__ == "__main__":
    chat_with_ai()


这段代码包含了：
1.  **错误处理**：捕获连接错误和限流，而不是报晦涩的Traceback。
2.  **流式响应**：文字会像打字机一样实时输出，体验更好。
3.  **无限对话**：支持你随时对话，输入 `exit` 退出。
